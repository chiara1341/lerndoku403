HTML

Elemente

Streng genommen besteht ein Element aus einem öffnenden und einem schliessenden Tag, und dem Inhalt, der dazwischensteht. Um einen öffnenden Tag korrekt zu schreiben verwendet man eine öffnende spitze Klammer(Kleiner-als-Zeichen) und eine schliessende spitze Klammer (Grösser-als-Zeichen). Zwischen diese beiden Klammern kommt ein Zeichen. Dieses bestimmt den Zweck des Tags. Ein schliessender Tag ist fast gleich aufgebaut mit dem Unterschied, dass nach der öffnenden spitzen Klammer ein Schrägstrich folgt /. Der Tag könnte folgendermassen aussehen: <h1> HTML </h1>

Jedes Element und jeder Text, der geschrieben wird befindet sich zwischen dem HTML-Element. Dieser Tag besagt, dass alles was zwischen dem Tag stehet HTML-Code ist. Auf den <html>-Tag folgt der <head> in welchem Informationen über die Seite stehen. Nach dem <head> folgt der <body>. Alles was sich innerhalb des <body>-Elements befindet wird auf dem Hauptfenster des Browsers angezeigt.
Für das Grundgerüst sind somit folgende Elemente notwendig:

  • <html>
  • <head>
  • <body>
Weitere Elemente, welche oft verwendet werden um die Übersicht einer Webseite zu fördern sind beispielsweise <header> und <footer>. Sie bilden einen Kopf- und Fussbereich oben und unten am Rand der jeweiligen Webseite.

Text

Um einen Text übersichtlicher erscheinen zu lassen ist es hilfreich verschiedene Überschriften zu verwenden. Eine Überschriften wird mit Hilfe des <h1>-Elements erstellt. Neben <h1> gibt es weitere Überschriften, welche der Nummern nach aufsteigen.
<h1> ist die Hauptüberschrift.
<h2> wird als Zwischenüberschrift verwendet.
Wird eine Überschrift benötigt, welche kleiner <h2> sein sollte wird <h3> und so weiter.
Der Browser zeigt die Überschrift der einzelnen Ebenen in verschiedenen Grössen an. <h1> ist am grössten und die darauffolgenden werden immer kleiner.
Um Ansätze zu erstellen, wie sie in einem normalen Textdokument mit Enter gemacht werden muss im HTML immer das Element <p>verwendet werden. Um eine Absatz zu erstellen, umgibt man die Worte, die ihn ausmachen, mit einem öffnenden <p>und einem schliessenden </p>-Tag.
Um Text im HTML zu formatieren gibt es einige Elemente. Grundsätzlich sollte dies jedoch im CSS gemacht werden.

<b> Fett
<i> Kursiv
<sup> Zahl hochstellen (Exponent)
<sub> Zahl tiefstellen
<br /> Um im Absatz einen Unterbruch zu verursachen und den Text auf eine neue Zeile schreiben
<hr /> Horizontale Linie zwischen den eingefügten Abschnitten
<strong> Text ist besonders wichtig und muss betont werden. Standartmässig zeigt der Browser den Text fett an.
<em> Eine Hervorbebung andeuten. Standartmässig zeigt der Browser den Text kursiv an.
<blockquote> Für ein Zitat, welches sich auf den ganzen Abschnitt bezieht.
<q> Für ein Zitat, welches innerhalb des Absatzes verwendet wird.
<cite> Um auf ein anders Werk zu verweisen. (Buch, Film, ...)
<address> Für Adresse. Sei es E-Mail oder Telefonnummer
<s> Kennzeichnet etwas, das nicht mehr stimmt aber trotzdem nicht gelöscht werden soll. Meist wird der Text durchgestrichen angezeigt.

Besonderes Markup

Doctype

Da es verschiedene Versionen von HTML gibt, sollte eine Webseite mit einer Dokumenttypdeklaration (DOCTYPE) beginnen, die dem Browser mitteilt, welche dieser Versionen verwendet wird.
Will man einen Kommentar verfassen, welchen die Benutzer der Webseite nicht sehen sollten, stellt man den Text zwischen folgende Zeichen: <!-- Hier kommentieren-->. Einen Code zu kommentieren ist sehr sinnvoll, denn es kann sein, dass man die Übersicht verliert und mithilfe der Kommentare ist das Verständnis klarer.

Attribut

Man kann jedem HTML-Element eine ID zuweisen. Dies führt dazu, dass das Element von den anderen Elementen unterscheidet werden kann. Wichtig ist, dass der Arttributwert mit einem Buchstaben oder Unterstrich beginnt. Ausserdem auf einer Seite zwei oder mehr Elemente den selben id Wert haben. Sonst ist der Wert nicht eindeutig.

Neben dem Attribut id gibt es ebenfalls das Attribut class. In manchen Situationen ist es notwendig mehrere Elemente zu markieren, um sie auf dieselbe Art und Weise von den restlichen Elementen abzuheben.

Block oder Inline-Element

Einige Elemente beginnen bei der Anzeige im Browserfenster immer in einer neuen Zeile. Solche Elemente werden Blockelemente genannt. Neben den Blockelementen gibt es ebenso Inline-Elemente. Inline-Elemente stehen in ein und derselben Zeile wie ihre Nachbarelemente.

Blockelement Inline-Element
<h1> <a>
<p> <b>
<ul> <em>
<li> <img>

Zu Blöcken gruppieren

Um mehrere Elemente zu einem Block zu gruppieren benötigt es das Element <div>. Wenn dieser <div> eine class- oder id-Attribut hinzugefügt wird, kann man CSS-Regeln erstellen. Durch die Gruppierung mit <div> werden die verschiedenen Elemente wie Blockelemente angeordnet. Nicht nur die Formatierung kann geändert werden, sondern auch die Übersichtlichkeit wird gefördert.

Zu Inline gruppieren

Mit dem Element <span>können Texte zu Inline-Elementen gruppieren. Sozusagen geschieht gerade das Gegenteil, wie wenn man Elemente mit <div> gruppiert

<iframe>

Das Verhalten eines iFrame, ist wie bei einem kleinen Fenster, welches in die Seite geschnitten wurde und in dem eine andere Seite sichtbar wird. iFrame ist die Abkürzung für Inline-Frame. Am häufigsten werden iFrames für Karten genutzt um sie von Google Maps einzubetten. Um einen iFrame einzubetten braucht es noch einige zusätzliche Attribute.

  • src
    Das Attribut src gibt den URL der Seite an, die in dem Frame angezeigt werden soll.
  • height
    Das Attribut height gibt die Höhe des iFrames in Pixeln an.
  • width
    Das Attribut width gibt die Breite des iFrames in Pixeln an.

Listen

Geordnete Listen

Geordnete Listen werden mit dem Element <ol> erstellt. Ein Eintrag muss zwischen den offenen Tag <li> und </li>geschrieben werden.
Dies könnte folgendermassen aussehen:

<ol>
<li>Aufstehen</li>
<li>Anziehen</li>
<li>Frühstücken</li>
<li>Zähne putzen</li>
</ol>
  1. Aufstehen
  2. Anziehen
  3. Frühstücken
  4. Zähne putzen

Ungeordnete Listen

Um eine ungeordnete Liste zu erstellen verwendet man das Element<ul>. Wie bei den geordneten Listen werden Listeneinträge mit <li> gemacht. Der Eintrag muss zwischen einem öffnenden Tag <li> und einem schliessendem </li> stehen. Der grundlegende Unterschied ist, das anstelle der Zahlen Aufzählungspunkte erscheinen

Definitionsliste

Eine weitere Variante um eine Liste zu erstellen ist die Definitionslist. Diese wird mit dem Element <dl> erstellt. Innerhalb dieses <dl>-Elements stehen normalerweise zwei weitere Elemente: <dt> und <dd>. Das <dt>-Element steht der Begriff, welcher erklärt werden muss. Der Tag <dd> enthaltet die Definition.

Verschachtelte Listen

Grundsätzlich sind verschachtelte Listen eine Liste, welche eine weitere Liste enthaltet. Verschachtelte Listen sind ungeordnete Listen.
Dies sieht folgendermassen aus:

<ul>
<li>Cremspeisen</li>
<li>Süssgebäck
<ul>
<li>Croissants</li>
<li>Profiteroles</li>
</ul>
</li>
</ul>
  • Cremspeisen
  • Süssgebäck
    • Croissants
    • Profiteroles

Formular

<form>

Formulare werden nicht nur für Suchvorgange verwendet, sondern auch wenn man sich auf einer Webseite neu registriert und ein Formular ausfüllen muss. Um ein Formularsteuerelement zu erstellen wird das Element <form>. Es muss über das Attribut action verfügen und normalerweise werden zusätzlich die Attribute method und id verwendet.
action: Der Wert von action ist die URL der Seite, auf den die Informationen gesendet werden, wenn ein Formular ausgefüllt wurde.
method: Die Art wie das Formular gesendet wird kann unterschieden werden. Zum einen gibt es get bei welchem die Informationen des Formulars am Ende der angegebenen URL angehängt werden. Dies ist ideal für kurze Formulare. Die andere Variante ist post. Die Werte des Formulars werden in HTTP-Headers gesendet. In Formularen, in denen man Dateien hochladen muss, lange Formulare und welchen mit sensiblen Daten. Wurde keine Methode angegeben werden die Informationenmit get gesendet.
id: Es wird für die Spezifikation des Wertes verwendet.

Texteingabe

Mit dem Tag <input> werden unterschiedliche Formularsteuerelement erstellt, deren Art durch den Wert des Attributs type bestimmt werden.
Hat das Attribut type den Wert text, wird ein einzeiliges Textfeld erstellt. Hat es den Wert password wird ein Textfeld erstellt, das die eingegebenen Zeichen nicht anzeigt und somit für Passwörter eignet.
Das Attribut name wird verwendet, dass der Server weiss, welches Formularsteuerelement zu welchen Daten gehören. Der Wert des Attributs ist das Steuerelement.
Mit dem Attribut maxlength kann angegeben werden, wie viele Zeichen maximal in ein Feld eingetragen werden dürfen.
Soll ein Textfeld erstellt werden, welches mehrere Zeilen enthalten sollte muss das Element <textarea> verwendet werden. Anders als die anderen Eingabeelemente verfügt diese Element über ein öffnendes und ein schliessendes Tag und ist somit kein leeres Element. Wird Text zwischen diese zwei Tags geschrieben wird dieser im Textfeld angezeigt, wenn die Seite ladet. Bevor ein Benutzer jedoch Text eingeben kann muss dieser den vorgegebenen Text löschen, sonst wird er mitgesendet.

Code:
<form action="http://www.example.com/login.php">

<p>Benutzername:
<input type="text" name="username" size="15" maxlength="30" />
</p>
</form>

Radio-Buttons

Hat das Attribut type den Wert radio werden Radio-Buttons erstellt, welche mehrere Optionen zur Auswahl geben. Mit dem Attribut value können die Auswahloptionen angegeben werden, welche an den Server gesendet werden. Das Attribut checked ist dafür zuständig, dass ausgewählt werden kann, welche Option beim Laden der Seite ausgewählt sein sollte. In einer Gruppe von Radio-Buttons kann höchstens einer dieses Attribut aufweisen.

Checkboxen

Neben den Radio-Buttons gibt es ebenfalls die Checkboxen. Der Unterschied ist, dass man bei Checkboxen mehrere Optionen wählen kann und es bei Radio-Buttons nur eine Option gibt. Die Checkbox wird mit dem Attribut type="checkbox" erstellt. Gleich wie bei den Radio-Buttons gibt es ebenfalls das Attribut checked um eine Vorauswahl zu treffen, wenn die Seite neu geladen wird. Um bei der gewünschten Option die Vorauswahl anzeigen zu lassen muss man dem Attribut checked den Wert checked geben. (checked="checked")

Dateieingabefeld

Ein Dateiupload ermöglicht den Benutzern, Dateien (z.B. Bilder) auf eine Webseite hochzuladen. Hierzu wird das Dateieingabefeld benötigt.
Wenn der Benutzer die Möglichkeit haben soll, eine Datei hochzuladen, braucht man das Dateieingabefeld, also ein <input>-Element. Das Attribut «type» mit dem Wert „file“ verursacht das erscheinen einer Schaltfläche zur Dateiauswahl. Klickt der Benutzer auf diese, öffnet sich ein Fenster, indem er die Dateien auswählen kann.
Mit dem type="submit" entsteht eine Sendeschaltfläche, die das Formular oder die Dateien an den Server überträgt. Die Schaltfläche kann mit dem Attribut «value» nach Belieben beschriftet werden.

Versteckte Steuerelemente

Das Element <button> wurde eingeführt, um der Schaltfläche ein spezielleres Erscheinungsbild zu geben, indem Text und Bild kombiniert werden.
Der <input> ist mit dem Attributwert type="hidden" angelegt. Das bedeutet, es ist ein verborgenes Formularsteuerelement enthalten, das nur für den Autor der Webseite sichtbar ist, nicht aber für den Benutzer.

Formularsteuerelemente beschriften

Formularsteuerelemente können mit dem <label>-Element beschriftet werden, sodass die Webseite auch für sehbehinderte Benutzer zugänglich ist. Dieses Element kann einerseits sowohl die Beschriftung, als auch das Steuerelement einschliessen (siehe Zeile 1 des Beispiels), andererseits kann es auch unabhängig von dem Steuerelement im Code stehen. Dann wird mit dem Attribut «for» angegeben, zu welchem Steuerelement die Beschriftung gehört (siehe Radio-Buttons). Der Wert des Attributs «for» ist der «id»-Wert des Steuerelements zu dem die Beschriftung gehört.

Formularvalidierung

Wird ein Formular falsch ausgefüllt, kann die Formularvalidierung dem Benutzer eine direkte Rückmeldung geben und so ein fehlerhaftes Absenden an den Server verhindert werden.

Datumeingaben

Für Datumseingaben kann, falls kein gewöhnliches Texteingabefeld verwendet werden soll, ein <input>-Element mit dem Wert „date“ für das Attribut «type» eingesetzt werden. Es entsteht dadurch ein Datumseingabefeld.

E-Mail-Adressen & URL

Für die Eingabe von E-Mail-Adressen und URLs kann durch ein passendes Eingabefeld mit dem <input>-Element und dem Attribut «type» eingerichtet werden. Durch eine HTML5-Validierung wird die Eingabe geprüft und falls sie im falschen Format vorliegt, gibt der Browser eine Fehlermeldung an den Benutzer weiter.
E-Mail: type="email"
URL: type="url"

Flash

Am häufigsten wird Flash mit JavaScript in Webseiten aufgenommen. Es gibt verschiedene Skripts, mit denen man dies tun können, ohne viel von JavaScript verstehen zu müssen.

Bilder

Um ein Bild zu einer Seite hinzuzufügen, verwendet man das Element <img>. Dies ist ein leeres Element. src, alt und title müssen als Attribut enthalten sein.
  • src: Das Attribut src wir für die Angabe der URL verwendet.
  • alt: In seltenen Fällen wird das Bild nicht angezeigt. Für solche Fälle wird mit dem Attribut alt das Bild in Text beschrieben. Es ist ebenfalls wichtig für die Suchmaschienen Optimierung und für Programme, welche die Webseite vorlesen, wenn jemand blind ist.
  • titel: Im Element <img> kann das Attribut titel verwendet werden, um zusätzliche Informationen über das Bild anzugeben.
Wird ein Bild vor dem Absatz platziert wird es vor dem Absatz angezeigt. Ist es am am Anfang des Absatzes platziert wird das Bild mit der unteren Kante auf der ersten Zeile platziert. Ist der Code des Bildes mitten im Abschnitt platziert erscheint es mitten in den Wörter.
